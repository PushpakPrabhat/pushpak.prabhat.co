Two years ago, I was a full-time animator at Unacademy. Today, I'm bridging the gap between animation and interactive technology at ET Money. Here's how that happened.

## The Pivot Point

It started with a simple observation: the best animations I was creating were going into static video players. Users watched them passively. But what if they could **interact** with the animation?

That question changed everything.

## Learning to Code

I started with JavaScript — not because I planned to become a developer, but because I wanted to make my animations interactive. Three nights a week after work, I'd study:

- **HTML/CSS** — The basics of the web
- **JavaScript** — Making things move programmatically
- **Three.js** — 3D graphics in the browser
- **GSAP** — Professional-grade animation library

## The Convergence

What I discovered is that animation and code are not two separate fields — **they're converging**. Tools like:

- **Lottie** — Export After Effects animations to the web
- **Rive** — Design interactive animations without code
- **Spline** — 3D design that exports to React components

...are proof that the future belongs to designers who can code and developers who can design.

## Advice for Animators

If you're an animator thinking about tech, here's my honest advice:

1. **Don't abandon your craft** — Your design eye is your superpower
2. **Start small** — Learn CSS animations before jumping to JavaScript
3. **Build projects** — The best way to learn is by building something you care about
4. **Share your journey** — Document everything, it helps others and reinforces your learning

The creative industry is changing. The question isn't whether to adapt — it's how fast you can do it.
